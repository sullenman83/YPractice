# YPractice
## Проектные работы в рамках обучения по программе "Продвинутая разработка на C# и .NET"

## В этой версии реализованы:
- REST API для работы с мероприятиями (CRUD). 
	- Операция получения всего спимка мероприятий
	- Операция получения мероприятия по id 
	- Операция создания нового мероприятия
	- Операция изменения мероприятия с заданным id
	- Операция удаления мероприятия по id
- Хранение мероприятий осуществляется в памяти
- Swagger для тестирования API
- Документация API


## Добавлена следующая функциональность
- Перехват необработанных исключений 
	Сообщения об ошибке выводятся в формате ProblemDetails (с заполнеными полями Title, Detail, Status)
- Добавлена фильтрация событий по следующим полям
	- title (string, опциональный) — поиск по названию (регистронезависимый, частичное совпадение);
	- from(DateTime, опциональный) — события, которые начинаются не раньше указанной даты;
	- to(DateTime, опциональный) — события, которые заканчиваются не позже указанной даты.
- Добавлена пагинация событий. Для разбивки по страницам служат следующие поля 
	- page (int, опциональный, c значением по умолчанию 1) — страница, которую необходимо вернуть;
	- pageSize (int, опциональный, со значением по умолчанию 10) — количество элементов на странице.
	

- Добавлены Unit тесты
	- Успешные сценарии:
		- создание события;
		- получение всех событий;
		- получение события по ID;
		- обновление существующего события;
		- удаление существующего события;
		- фильтрация по названию;
		- фильтрация по датам (startDate, endDate);
		- пагинация событий;
		- комбинированная фильтрация.

	- Неуспешные сценарии:
		- попытка получить событие с несуществующим ID;
		- попытка обновить событие с несуществующим ID;
		- создание события с некорректными данными (если валидация в сервисе);
		- обновление события с некорректными датами (EndAt раньше StartAt).
